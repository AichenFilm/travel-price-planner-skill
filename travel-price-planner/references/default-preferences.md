# Default Preferences Template

Use this file as a neutral preference template. Do not assume these values are true for a user; collect or infer them from the request.

## Preference Fields To Ask Only When Needed

- Home region and usual departure airports or stations.
- Acceptable destination airports, stations, neighborhoods, or nearby alternatives.
- Date flexibility and whether overnight travel is acceptable.
- Number of travelers, ages if relevant, and cabin/class preference.
- Checked baggage, carry-on baggage, seat, meal, breakfast, or accessibility needs.
- Refund/change/cancellation tolerance.
- Budget ceiling and preferred currency.
- Passport, visa, residence permit, transit, or entry constraints.
- Payment method or platform preference when it materially affects price or support.

## General Defaults

- Prefer lowest reasonable total cost, balancing money, time, risk, aftersales, and flexibility.
- Prefer official booking when the official price is within about 30-50 units of the chosen base currency for medium/long-haul travel.
- Treat familiar OTAs as useful comparison and convenience platforms, not automatically better booking channels.
- Treat metasearch tools as route and price discovery tools, not the booking or aftersales party.
- Do not assume the user accepts self-transfer, separate tickets, nonrefundable rates, login-only discounts, or risky transit.

## Ask Only When Needed

Ask for clarification only when the missing detail materially changes the result, such as passport/visa constraints, strict arrival deadlines, checked baggage needs, hotel cancellation needs, or whether a risky cheap option is acceptable.
